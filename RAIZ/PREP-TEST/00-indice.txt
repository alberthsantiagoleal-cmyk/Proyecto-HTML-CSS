REGLA GENERAL PARA CUALQUIER ESCENARIO
----------------------------------------------------------------

  1. Crear el HTML en SCREENS/ con la misma estructura base:
       <!DOCTYPE html> + <head> con app.css + <section id="...">

  2. Reutilizar las clases que ya existen:
       .dest-list / .dest-item   → cualquier lista
       .home-header              → encabezado con flecha atrás
       .btn-confirm              → botón dorado de acción
       .btn-add                  → botón secundario gris
       .pay-back                 → flecha de regreso

  3. Agregar los estilos nuevos AL FINAL de app.css,
     nunca en medio del archivo.

  4. Enlazar la pantalla nueva desde la pantalla que
     tenga más sentido (home, nav, banner, etc.)

----------------------------------------------------------------
TÉCNICAS CSS DEL PROYECTO (para explicar en la sustentación)
----------------------------------------------------------------

  :active          → efecto press en categorías (scale)
  :focus-within    → borde dorado en barra de búsqueda
  :hover           → opacidad en listas, tooltips
  :checked + ~     → selección de naves, métodos de pago,
                     filtros, modo claro
  :checked + +     → acordeón FAQ (hermano adyacente)
  :target          → modal de agregar tarjeta
  @keyframes       → animaciones splash (splashDrop)
                     y receipt (fadeUp)
  CSS variables    → sistema de diseño centralizado en :root
  transition       → todas las animaciones de estado

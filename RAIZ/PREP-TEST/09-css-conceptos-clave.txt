================================================================
CSS — CONCEPTOS CLAVE PARA LA SUSTENTACIÓN
================================================================

----------------------------------------------------------------
1. VARIABLES CSS (:root)
----------------------------------------------------------------
Qué son:
  Valores reutilizables definidos una sola vez al inicio del CSS.
  Se usan en todo el archivo con var(--nombre).

Por qué importan:
  Si el profesor pide cambiar el color dorado del proyecto,
  solo tocas UNA línea: --accent: #f0a500
  Y cambia en todos los botones, bordes y marcas automáticamente.

Ejemplo en el proyecto:
  --accent: #f0a500   → color dorado de todos los elementos activos
  --bg: #0d1117       → fondo oscuro de todas las pantallas
  --muted: #8b949e    → color gris de textos secundarios
  --rs: 12px          → radio de borde de todas las tarjetas

----------------------------------------------------------------
2. PSEUDO-CLASE :checked
----------------------------------------------------------------
Qué es:
  Se activa cuando un input (radio o checkbox) está marcado.

Cómo se usa en el proyecto:
  Los inputs están OCULTOS con display:none.
  Los labels son lo que el usuario ve.
  Cuando el usuario hace clic en el label, el input se marca,
  y CSS usa :checked para cambiar el estilo del label.

Selector clave — el símbolo ~:
  #ship-1:checked ~ label[for="ship-1"]
  Significa: "cuando ship-1 esté marcado, aplica esto
  al label que tenga for='ship-1' y sea hermano en el HTML"

Dónde se usa:
  - Selección de naves (03-travel)
  - Selección de método de pago (05-payment)
  - Botón swipe de confirmación (04-booking)

----------------------------------------------------------------
3. PSEUDO-CLASE :target
----------------------------------------------------------------
Qué es:
  Se activa cuando el id de un elemento coincide con
  el hash (#) que hay en la URL del navegador.

Cómo se usa en el proyecto:
  El botón "Agregar nuevo método" tiene href="#modal-add".
  Al hacer clic, la URL pasa a ser ...05-payment.html#modal-add
  CSS detecta eso con .modal-overlay:target y muestra el modal.
  El botón ✕ tiene href="#" que limpia el hash y cierra el modal.

Por qué es importante:
  Es la única forma de abrir/cerrar un modal sin JavaScript.

----------------------------------------------------------------
4. PSEUDO-CLASE :focus-within
----------------------------------------------------------------
Qué es:
  Se activa en un elemento padre cuando CUALQUIER hijo
  dentro de él está enfocado (recibe el cursor).

Cómo se usa en el proyecto:
  La barra de búsqueda es un label que contiene un input.
  Cuando el usuario hace clic en el input, el label detecta
  el foco con :focus-within y cambia su borde a dorado.

  .search-bar:focus-within { border-color: var(--accent); }

----------------------------------------------------------------
5. PSEUDO-CLASE :active
----------------------------------------------------------------
Qué es:
  Se activa mientras el usuario mantiene presionado un elemento.

Cómo se usa en el proyecto:
  Las categorías de Home tienen clase .pressable.
  Al presionarlas se encogen con transform: scale(0.93),
  simulando el feedback táctil de una app móvil real.

----------------------------------------------------------------
6. @keyframes (ANIMACIONES)
----------------------------------------------------------------
Qué son:
  Definen el movimiento de una animación paso a paso.
  Se aplican a elementos con la propiedad animation.

Las dos animaciones del proyecto:

  splashDrop → el logo del splash cae desde arriba
    from: opacity 0, translateY(-24px)  → invisible, arriba
    to:   opacity 1, translateY(0)      → visible, en su lugar

  fadeUp → elementos del receipt suben desde abajo
    from: opacity 0, translateY(20px)   → invisible, abajo
    to:   opacity 1, translateY(0)      → visible, en su lugar

animation-delay:
  Cada elemento del receipt tiene un delay diferente
  (0.1s, 0.2s, 0.3s...) para que aparezcan en secuencia,
  no todos al mismo tiempo.

----------------------------------------------------------------
7. TRANSITION
----------------------------------------------------------------
Qué es:
  Hace que un cambio de estilo ocurra gradualmente
  en lugar de ser instantáneo.

Cómo se usa en el proyecto:
  Casi todos los elementos interactivos tienen:
  transition: background 0.2s ease
  transition: opacity 0.3s ease
  transition: transform 0.3s ease

  El swipe-link usa dos propiedades a la vez:
  transition: opacity 0.35s ease, transform 0.35s ease

----------------------------------------------------------------
8. POSITION: ABSOLUTE y CAPAS (z-index)
----------------------------------------------------------------
Qué es:
  Saca un elemento del flujo normal y lo ubica en
  coordenadas exactas dentro de su padre.

Cómo se usa en el proyecto:
  El mapa espacial tiene 5 capas apiladas:
    z-index 0 → .space-map-bg  (fondo negro)
    z-index 0 → .stars         (estrellas CSS)
    z-index 2 → .orbit-path    (línea de órbita)
    z-index 5 → .planet-mars   (planeta destino)
    z-index 5 → .planet-earth  (planeta origen)
    z-index 10 → .map-back     (flecha de regreso, siempre encima)

----------------------------------------------------------------
9. FLEXBOX
----------------------------------------------------------------
Qué es:
  Sistema de layout que organiza elementos en fila o columna
  y los alinea automáticamente.

Cómo se usa en el proyecto:
  Casi todo el layout usa flexbox:
  - flex-direction: column → apila elementos verticalmente
  - align-items: center    → centra horizontalmente
  - justify-content: space-between → separa elementos a los extremos
  - flex: 1                → hace que un elemento ocupe el espacio restante
  - gap: 12px              → espacio entre elementos sin usar margin

----------------------------------------------------------------
10. SELECTOR + (HERMANO ADYACENTE)
----------------------------------------------------------------
Qué es:
  Selecciona el elemento que viene INMEDIATAMENTE después.

Cómo se usa en el proyecto:
  El swipe usa tres elementos seguidos en el HTML:
    input#swipe-check
    label.swipe-btn
    a.swipe-link

  CSS los encadena así:
  .swipe-check:checked + .swipe-btn         → cambia el botón
  .swipe-check:checked + .swipe-btn + .swipe-link → muestra el enlace

  El + significa "el hermano que viene justo después".
